Chapter 13: Managing Schema Objects

ALTER TABLE and schema object management

1. ALTER TABLE Statement

Use ALTER TABLE to modify existing table structure:

  • Add new columns
  • Modify existing columns
  • Define default values
  • Drop columns
-- Add column ALTER TABLE emp_master ADD (date_of_birth DATE); -- Add multiple columns ALTER TABLE emp_master ADD (phone VARCHAR2(15), email VARCHAR2(100)); -- Modify column ALTER TABLE emp_master MODIFY (ename VARCHAR2(50)); -- Drop column ALTER TABLE emp_master DROP COLUMN date_of_birth;

2. Adding Columns

-- Add column with default value ALTER TABLE emp_master ADD (status VARCHAR2(10) DEFAULT 'ACTIVE'); -- Add NOT NULL column (only if table is empty) ALTER TABLE empty_table ADD (required_field VARCHAR2(20) NOT NULL);
Important: You cannot add a NOT NULL column to a table that contains data unless you provide a default value.

3. Modifying Columns

-- Increase column size ALTER TABLE emp_master MODIFY (ename VARCHAR2(100)); -- Change data type (if compatible) ALTER TABLE emp_master MODIFY (empno NUMBER(10)); -- Add NOT NULL constraint ALTER TABLE emp_master MODIFY (ename NOT NULL);

4. Managing Constraints

-- Add constraint ALTER TABLE emp_master ADD CONSTRAINT emp_pk PRIMARY KEY (empno); -- Add foreign key ALTER TABLE emp_master ADD CONSTRAINT emp_dept_fk FOREIGN KEY (deptno) REFERENCES dept(deptno); -- Drop constraint ALTER TABLE emp_master DROP CONSTRAINT emp_pk; -- Disable constraint ALTER TABLE emp_master DISABLE CONSTRAINT emp_dept_fk; -- Enable constraint ALTER TABLE emp_master ENABLE CONSTRAINT emp_dept_fk;

5. Best Practices

  • Plan Changes: Test ALTER statements on development first
  • Backup Data: Always backup before major changes
  • Check Dependencies: Verify impact on views, procedures
  • Monitor Performance: Large tables may take time to alter
