Chapter 15: Managing Data in Different Timezones

Working with timezone-aware data types and functions

1. Timezone Data Types

Data Type Description
TIMESTAMP Date and time with fractional seconds
TIMESTAMP WITH TIME ZONE TIMESTAMP with timezone offset
TIMESTAMP WITH LOCAL TIME ZONE TIMESTAMP normalized to database timezone
-- Create table with timezone columns CREATE TABLE events ( id NUMBER, event_name VARCHAR2(100), event_time TIMESTAMP, event_time_tz TIMESTAMP WITH TIME ZONE, event_time_ltz TIMESTAMP WITH LOCAL TIME ZONE ); -- Insert timezone data INSERT INTO events VALUES ( 1, 'Conference Call', TIMESTAMP '2023-12-25 14:30:00.123', TIMESTAMP '2023-12-25 14:30:00.123 -05:00', TIMESTAMP '2023-12-25 14:30:00.123' );

2. Timezone Functions

-- Current timestamp functions SELECT CURRENT_TIMESTAMP, LOCALTIMESTAMP, SYSTIMESTAMP FROM dual; -- Convert between timezones SELECT SYSTIMESTAMP, SYSTIMESTAMP AT TIME ZONE 'US/Eastern' as eastern_time, SYSTIMESTAMP AT TIME ZONE 'Europe/London' as london_time FROM dual; -- Extract timezone information SELECT EXTRACT(TIMEZONE_HOUR FROM SYSTIMESTAMP) tz_hour, EXTRACT(TIMEZONE_MINUTE FROM SYSTIMESTAMP) tz_minute FROM dual;

3. Working with Timezones

-- Convert timestamp to different timezone SELECT event_name, event_time_tz, event_time_tz AT TIME ZONE 'US/Pacific' as pacific_time, event_time_tz AT TIME ZONE 'Asia/Tokyo' as tokyo_time FROM events; -- Compare times across timezones SELECT event_name, event_time_tz, CASE WHEN event_time_tz AT TIME ZONE 'UTC' > SYSTIMESTAMP AT TIME ZONE 'UTC' THEN 'Future Event' ELSE 'Past Event' END as event_status FROM events;

4. Timezone Best Practices

  • Store in UTC: Use TIMESTAMP WITH LOCAL TIME ZONE for global applications
  • Display in User Timezone: Convert for presentation
  • Validate Timezones: Check timezone names and offsets
  • Handle DST: Account for daylight saving time changes
-- Query timezone names SELECT * FROM V$TIMEZONE_NAMES WHERE TZNAME LIKE 'US/%'; -- Set session timezone ALTER SESSION SET TIME_ZONE = 'US/Eastern';
