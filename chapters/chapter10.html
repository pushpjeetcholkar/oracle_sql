Chapter 10: Creating Other Schema Objects

Views, sequences, indexes, and synonyms

1. Schema Objects Overview

Object Purpose
View Logical representation of data from tables
Sequence Generate unique numerical values
Index Improve query performance
Synonym Provide alternate names for objects

2. Views

A view is a logical table based on one or more tables. It contains no data but acts like a window to view or modify table data.

View Advantages:

  • Restrict Data Access: Show only selected columns
  • Simplify Complex Queries: Hide join complexity
  • Data Independence: Access multiple tables through one view
  • Different Perspectives: Present same data differently
-- Create simple view CREATE VIEW emp_view AS SELECT empno, ename, job, sal FROM emp WHERE deptno = 10; -- Create complex view with joins CREATE VIEW emp_dept_view AS SELECT e.empno, e.ename, e.sal, d.dname, d.loc FROM emp e JOIN dept d ON e.deptno = d.deptno; -- Use view like a table SELECT * FROM emp_view WHERE sal > 2000;

3. Sequences

Sequences generate unique numbers automatically, commonly used for primary keys.

-- Create sequence CREATE SEQUENCE emp_seq START WITH 1000 INCREMENT BY 1 MAXVALUE 9999 NOCACHE NOCYCLE; -- Use sequence INSERT INTO emp VALUES (emp_seq.NEXTVAL, 'JOHN', 'CLERK', 7902, SYSDATE, 1200, NULL, 20); -- Get current value SELECT emp_seq.CURRVAL FROM dual; -- Drop sequence DROP SEQUENCE emp_seq;

Sequence Options:

  • START WITH: Starting value
  • INCREMENT BY: Step value
  • MAXVALUE/NOMAXVALUE: Maximum value
  • MINVALUE/NOMINVALUE: Minimum value
  • CYCLE/NOCYCLE: Restart when limit reached
  • CACHE/NOCACHE: Pre-allocate numbers in memory

4. Indexes

Indexes improve query performance by creating faster access paths to table data.

-- Create index on single column CREATE INDEX emp_ename_idx ON emp(ename); -- Create composite index CREATE INDEX emp_dept_job_idx ON emp(deptno, job); -- Create unique index CREATE UNIQUE INDEX emp_email_idx ON emp(email); -- Drop index DROP INDEX emp_ename_idx;

Index Types:

  • B-tree Index: Default type, good for equality and range queries
  • Unique Index: Enforces uniqueness
  • Composite Index: Multiple columns
  • Function-based Index: Based on expressions

5. Synonyms

Synonyms provide alternative names for database objects.

-- Create public synonym CREATE PUBLIC SYNONYM employees FOR hr.emp; -- Create private synonym CREATE SYNONYM e FOR emp; -- Use synonym SELECT * FROM e WHERE deptno = 10; -- Drop synonym DROP SYNONYM e; DROP PUBLIC SYNONYM employees;

Synonym Benefits:

  • Hide object location and ownership
  • Provide shorter names for long object names
  • Enable location transparency
  • Simplify SQL statements
